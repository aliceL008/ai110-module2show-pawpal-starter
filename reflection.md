# PawPal+ Project Reflection

**What are the limitations or biases in your system?**

The main limitation is that the knowledge base focuses on common pets like dogs and cats, so rare or exotic species get fewer KB matches and rely more heavily on live You.com searches. The search itself is filtered to English-language vet sites based in the US and UK, which means the recommendations may not reflect care standards from other regions or cultures. Another bias is that content retrieved from websites is only kept if it matches a predefined set of topic keywords defined in the code, such as "feed", "exercise", "groom", "dental", and "train". This means any relevant care advice that uses different wording or covers topics outside that list will be silently ignored, even if the source is trustworthy — limiting the range of information the system can surface. Task durations are extracted from text snippets using pattern matching, so they can occasionally be off if a source phrases things unusually. There is also no veterinary verification — the system trusts its sources but cannot guarantee medical accuracy, which is why it includes confidence scores and cited reasoning so users can judge for themselves.

**Could your AI be misused, and how would you prevent that?**

Someone could follow the generated schedule as if it were a substitute for actual veterinary advice, especially for a sick or injured pet. The schedule is designed for routine daily care, not medical diagnosis or treatment. To prevent misuse, the app should include a clear disclaimer that recommendations are based on general vet guidelines and are not a replacement for a licensed veterinarian. The confidence scorer already signals when output is less reliable, and the sourced reasoning lets users verify where information came from. Adding a visible notice on the UI that flags health concerns beyond the scope of the schedule would further reduce the risk of misuse.

**What surprised you while testing your AI's reliability?**

The biggest surprise was how much noise the You.com API returned before filtering was added. General search queries for breeds like "Golden Retriever care" came back with ad snippets, gift card promotions, and lifestyle content that had nothing to do with pet health. It was unexpected that a search platform would surface that kind of content for a medical topic. The second surprise was how well the fallback system performed even without an AI language model — the vet-sourced knowledge base alone produced reasonable, cited recommendations that passed all 30 tests. This showed that a well-curated retrieval layer can carry a lot of the load that is typically expected from a generative model.

**Describe your collaboration with AI during this project. Identify one instance when the AI gave a helpful suggestion and one instance where its suggestion was flawed or incorrect.**

AI was used throughout this project for debugging, writing tests, building the Streamlit UI, and expanding the knowledge base. One instance where it gave genuinely helpful suggestions was in optimizing the retrieval system — it proposed different approaches for building better tests to cover edge cases like unknown species, extreme ages, and empty breed fields, and suggested specific filter keywords to narrow search results to only relevant care topics. Those suggestions directly improved both the reliability of the test suite and the quality of the content being retrieved. One instance where its suggestions were flawed was when it kept generating hardcoded pet care data based on its own training knowledge instead of using the You.com API to pull real content from websites. When asked to generate recommendations for different species, it would return the same generic information regardless of the breed or species entered, rather than searching for and surfacing distinct, source-specific guidelines. It took identifying the pattern, questioning the approach, and explicitly asking it to pull from different sources for each species before it started returning diverse and relevant results. It was a reminder that AI defaults to what it already knows and needs to be actively redirected when the goal is to retrieve real, varied, and up-to-date information.
